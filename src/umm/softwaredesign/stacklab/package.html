Contains an interface and implementation of a basic stack class. Used in lab exercises in the University of Minnesota, Morris, CSci 3601 (Software Design and Development) course.

The test directory contains a suite of specifications written using the Spock specification framework that defines the expected behavior of the Stack class. When you get that suite of specifications to pass, then you probably have a pretty reasonable implementation of of the Stack class, although it's always possible that there can be errors that aren't caught by these specifications, so don't just assume that your implementation is completely correct just because the tests pass

Package Specification

  • StackIF is the interface that defines what a stack should be.
  • Stack should (eventually) become a complete implementation of that interface.
  • StackUnderflowException is a runtime exception that is thrown when a stack underflows, i.e., when we pop or top an empty stack.
